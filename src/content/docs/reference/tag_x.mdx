`_Nx` multiplies the source value by `N` and formats the result with the source's default decimal precision and unit.

`N` can be any integer or decimal number: `_2x`, `_0.5x`, `_1.5x`, etc.

For example, for a source with value `8.30V` (Lipo2S):
- `_2x` outputs `16.60`
- `_0.5x` outputs `4.15`

A common use case is displaying a cell voltage from a pack voltage.

:::note
Unlike `_Nv` which controls decimal places, `_Nx` scales the value while keeping the source's original formatting.
:::
